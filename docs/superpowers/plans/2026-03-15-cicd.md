# CI/CD Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Set up GitHub Actions CI/CD with OIDC authentication to build/test the backend on PRs and deploy all three Lambda containers to ECR on merge to main, with Terraform plan-as-comment on infra PRs and auto-apply on merge.

**Architecture:** Two workflow files with path filters (`backend.yml`, `infra.yml`). A new `github-oidc` Terraform module creates the IAM OIDC provider and a single CI role (scoped to the repo). Backend workflow runs `./gradlew build test` on PRs; on merge builds three Docker images, pushes to ECR, and updates Lambda `prod` aliases. Infra workflow runs `terraform validate + plan` on PRs (posts plan as comment), and `terraform apply` on merge. Frontend is auto-deployed by Amplify — no workflow needed.

**Tech Stack:** GitHub Actions, `aws-actions/configure-aws-credentials@v4` (OIDC), `aws-actions/amazon-ecr-login@v2`, `hashicorp/setup-terraform@v3`, `actions/github-script@v7`, Terraform ~> 1.14, AWS CLI v2, Docker, Gradle 9, Java 25 (Temurin).

**Spec:** `docs/superpowers/specs/2026-03-15-stanford-course-finder-design.md` §12

- [ ] **Step 2: Create `main.tf`**

```hcl
# infra/modules/github-oidc/main.tf

data "aws_caller_identity" "current" {}

# One OIDC provider per AWS account — skip if it already exists
resource "aws_iam_openid_connect_provider" "github" {
  count           = var.create_oidc_provider ? 1 : 0
  url             = "https://token.actions.githubusercontent.com"
  client_id_list  = ["sts.amazonaws.com"]
  thumbprint_list = ["6938fd4d98bab03faadb97b34396831e3780aea1"]

  lifecycle {
    # AWS rotates thumbprints; ignore drift after initial creation
    ignore_changes = [thumbprint_list]
  }
}

locals {
  oidc_provider_arn = (var.create_oidc_provider
    ? aws_iam_openid_connect_provider.github[0].arn
    : "arn:aws:iam::${data.aws_caller_identity.current.account_id}:oidc-provider/token.actions.githubusercontent.com"
  )
}

resource "aws_iam_role" "ci" {
  name = "stanford-courses-ci"

  assume_role_policy = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Effect    = "Allow"
      Principal = { Federated = local.oidc_provider_arn }
      Action    = "sts:AssumeRoleWithWebIdentity"
      Condition = {
        StringLike = {
          "token.actions.githubusercontent.com:sub" = "repo:${var.github_org}/${var.github_repo}:*"
        }
        StringEquals = {
          "token.actions.githubusercontent.com:aud" = "sts.amazonaws.com"
        }
      }
    }]
  })
}

resource "aws_iam_role_policy_attachment" "ci_admin" {
  role       = aws_iam_role.ci.name
  policy_arn = "arn:aws:iam::aws:policy/AdministratorAccess"
}
```

## Chunk 2: GitHub Actions Workflows

### Task 3: Backend CI/CD Workflow

**Files:**

- Create: `.github/workflows/backend.yml`

This workflow:

- On **PR** touching `backend/`: runs `./gradlew build test` (required status check)
- On **push to main** touching `backend/`: builds three Docker images, pushes to ECR, updates Lambda `prod` aliases

Lambda function names match what the infra plan creates: `stanford-courses-api`, `stanford-courses-ingestion`, `stanford-courses-post-confirmation`.

- [ ] **Step 1: Create `.github/workflows/` directory and `backend.yml`**

```bash
mkdir -p .github/workflows
```

- [ ] **Step 2: Write `backend.yml`**

```yaml
# .github/workflows/backend.yml
name: Backend

on:
  push:
    branches: [main]
    paths: ['backend/**']
  pull_request:
    paths: ['backend/**']

permissions:
  id-token: write   # required for OIDC
  contents: read

env:
  AWS_REGION: us-east-1
  PROJECT: stanford-courses

jobs:
  test:
    name: Build & Test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '25'
          cache: gradle

      - name: Build and test all modules
        run: cd backend && ./gradlew build test

  deploy:
    name: Deploy Lambdas
    needs: test
    if: github.ref == 'refs/heads/main' && github.event_name == 'push'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0   # needed for git tag lookup

      - uses: actions/setup-java@v4
        with:
          distribution: temurin
          java-version: '25'
          cache: gradle

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ secrets.AWS_OIDC_ROLE_ARN }}
          aws-region: ${{ env.AWS_REGION }}

      - name: Login to Amazon ECR
        id: ecr
        uses: aws-actions/amazon-ecr-login@v2

      - name: Compute image tag
        id: tag
        run: |
          GIT_SHA=$(git rev-parse --short HEAD)
          GIT_TAG=$(git tag --points-at HEAD | grep '^v' | head -1 || true)
          if [ -n "$GIT_TAG" ]; then
            echo "IMAGE_TAG=${GIT_TAG}-${GIT_SHA}" >> "$GITHUB_OUTPUT"
          else
            echo "IMAGE_TAG=${GIT_SHA}" >> "$GITHUB_OUTPUT"
          fi

      - name: Build fat JARs
        run: |
          cd backend
          ./gradlew :api:shadowJar :ingestion:shadowJar :post-confirmation:shadowJar

      - name: Build and push API image
        env:
          REGISTRY: ${{ steps.ecr.outputs.registry }}
          TAG: ${{ steps.tag.outputs.IMAGE_TAG }}
        run: |
          docker build -t "$REGISTRY/${{ env.PROJECT }}-api:$TAG" backend/api/
          docker push "$REGISTRY/${{ env.PROJECT }}-api:$TAG"

      - name: Build and push Ingestion image
        env:
          REGISTRY: ${{ steps.ecr.outputs.registry }}
          TAG: ${{ steps.tag.outputs.IMAGE_TAG }}
        run: |
          docker build -t "$REGISTRY/${{ env.PROJECT }}-ingestion:$TAG" backend/ingestion/
          docker push "$REGISTRY/${{ env.PROJECT }}-ingestion:$TAG"

      - name: Build and push Post-Confirmation image
        env:
          REGISTRY: ${{ steps.ecr.outputs.registry }}
          TAG: ${{ steps.tag.outputs.IMAGE_TAG }}
        run: |
          docker build -t "$REGISTRY/${{ env.PROJECT }}-post-confirmation:$TAG" backend/post-confirmation/
          docker push "$REGISTRY/${{ env.PROJECT }}-post-confirmation:$TAG"

      - name: Deploy API Lambda
        env:
          REGISTRY: ${{ steps.ecr.outputs.registry }}
          TAG: ${{ steps.tag.outputs.IMAGE_TAG }}
        run: |
          aws lambda update-function-code \
            --function-name ${{ env.PROJECT }}-api \
            --image-uri "$REGISTRY/${{ env.PROJECT }}-api:$TAG"
          aws lambda wait function-updated-v2 \
            --function-name ${{ env.PROJECT }}-api
          NEW_VERSION=$(aws lambda publish-version \
            --function-name ${{ env.PROJECT }}-api \
            --query Version --output text)
          aws lambda update-alias \
            --function-name ${{ env.PROJECT }}-api \
            --name prod \
            --function-version "$NEW_VERSION"

      - name: Deploy Ingestion Lambda
        env:
          REGISTRY: ${{ steps.ecr.outputs.registry }}
          TAG: ${{ steps.tag.outputs.IMAGE_TAG }}
        run: |
          aws lambda update-function-code \
            --function-name ${{ env.PROJECT }}-ingestion \
            --image-uri "$REGISTRY/${{ env.PROJECT }}-ingestion:$TAG"
          aws lambda wait function-updated-v2 \
            --function-name ${{ env.PROJECT }}-ingestion
          NEW_VERSION=$(aws lambda publish-version \
            --function-name ${{ env.PROJECT }}-ingestion \
            --query Version --output text)
          aws lambda update-alias \
            --function-name ${{ env.PROJECT }}-ingestion \
            --name prod \
            --function-version "$NEW_VERSION"

      - name: Deploy Post-Confirmation Lambda
        env:
          REGISTRY: ${{ steps.ecr.outputs.registry }}
          TAG: ${{ steps.tag.outputs.IMAGE_TAG }}
        run: |
          aws lambda update-function-code \
            --function-name ${{ env.PROJECT }}-post-confirmation \
            --image-uri "$REGISTRY/${{ env.PROJECT }}-post-confirmation:$TAG"
          aws lambda wait function-updated-v2 \
            --function-name ${{ env.PROJECT }}-post-confirmation
          NEW_VERSION=$(aws lambda publish-version \
            --function-name ${{ env.PROJECT }}-post-confirmation \
            --query Version --output text)
          aws lambda update-alias \
            --function-name ${{ env.PROJECT }}-post-confirmation \
            --name prod \
            --function-version "$NEW_VERSION"
```

### Task 4: Infra CI/CD Workflow

**Files:**

- Create: `.github/workflows/infra.yml`

This workflow:

- On **PR** touching `infra/`: runs `terraform validate` + `terraform plan`, posts plan output as a PR comment
- On **push to main** touching `infra/`: runs `terraform apply -auto-approve`

- [ ] **Step 1: Write `infra.yml`**

```yaml
# .github/workflows/infra.yml
name: Infrastructure

on:
  push:
    branches: [main]
    paths: ['infra/**']
  pull_request:
    paths: ['infra/**']

permissions:
  id-token: write       # OIDC
  contents: read
  pull-requests: write  # post plan as comment

env:
  AWS_REGION: us-east-1
  TF_VERSION: "1.14.0"

jobs:
  plan:
    name: Terraform Plan
    if: github.event_name == 'pull_request'
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: infra
    steps:
      - uses: actions/checkout@v4

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ secrets.AWS_OIDC_ROLE_ARN }}
          aws-region: ${{ env.AWS_REGION }}

      - uses: hashicorp/setup-terraform@v3
        with:
          terraform_version: ${{ env.TF_VERSION }}

      - name: Terraform Init
        run: terraform init

      - name: Terraform Validate
        run: terraform validate

      - name: Terraform Plan
        id: plan
        run: terraform plan -no-color 2>&1 | tee /tmp/plan_output.txt
        continue-on-error: true

      - name: Post plan as PR comment
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const plan = fs.readFileSync('/tmp/plan_output.txt', 'utf8');
            const MAX = 60000;
            const body = plan.length > MAX
              ? plan.substring(0, MAX) + '\n...(truncated — see Actions log for full output)'
              : plan;
            await github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: `## Terraform Plan\n\`\`\`\n${body}\n\`\`\``
            });

      - name: Fail job if plan failed
        if: steps.plan.outcome == 'failure'
        run: exit 1

  apply:
    name: Terraform Apply
    if: github.ref == 'refs/heads/main' && github.event_name == 'push'
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: infra
    steps:
      - uses: actions/checkout@v4

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ secrets.AWS_OIDC_ROLE_ARN }}
          aws-region: ${{ env.AWS_REGION }}

      - uses: hashicorp/setup-terraform@v3
        with:
          terraform_version: ${{ env.TF_VERSION }}

      - name: Terraform Init
        run: terraform init

      - name: Terraform Apply
        run: terraform apply -auto-approve
```
